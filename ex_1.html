🤖 Arduino Introduction

Arduino is an open-source hardware and software platform for building digital devices. The Arduino Uno uses the ATmega328P microcontroller running at 16 MHz with 2 KB RAM and 32 KB flash.

💡
Microcontroller = All-in-one chip

Arduino Uno is a microcontroller — all components (CPU, RAM, Flash, I/O) are on a single chip. No OS needed. Runs a single dedicated program.

Applications

  • Home automation & smart systems
  • Robotics & motor control
  • IoT & environmental monitoring (with WiFi shield)
  • Security systems, weather monitoring, smart agriculture

📸 Arduino Uno Board Diagram

Arduino Uno Board — showing digital pins (D0–D13), analog pins (A0–A5), power pins, I2C (A4=SDA, A5=SCL), and UART (TX=1, RX=0)
🎯
Key Pins to Memorize

A4 = SDA (I2C Data for OLED)  |  A5 = SCL (I2C Clock for OLED)
A0–A5 = Analog input (for MQ-2 AO pin)  |  D0(RX)/D1(TX) = UART Serial
D13 = Built-in LED  |  D3,5,6,9,10,11 = PWM pins (marked with ~)

No other code structure will work. These two functions are the skeleton of every sketch.

  • Called once when the Arduino powers on or is reset

⬡ Arduino C++
setup() example
void setup() { pinMode(13, OUTPUT); // configure pin 13 as output Serial.begin(9600); // start serial communication // This block runs ONCE only! }

loop() example
void loop() { digitalWrite(13, HIGH); // LED ON delay(1000); // wait 1 second digitalWrite(13, LOW); // LED OFF delay(1000); // wait 1 second // automatically repeats forever! }

🔑 Key Functions Deep Dive — High Exam Priority

These are the most commonly tested functions. Know what each does, its syntax, and example.

1. digitalWrite(pin, value) — Output to a pin

⚡
What it does

Sets a digital pin to either HIGH (5V) or LOW (0V). Used to turn on/off LEDs, relays, buzzers, etc. The pin must first be set as OUTPUT using pinMode().

⬡ Arduino
digitalWrite
digitalWrite(13, HIGH); // Pin 13 → 5V → LED turns ON digitalWrite(13, LOW); // Pin 13 → 0V → LED turns OFF digitalWrite(1, HIGH); // Pin 1 → 5V (can use any digital pin)

2. digitalRead(pin) — Read a digital signal

📖
What it does

Reads the state of a digital pin — returns either HIGH (5V/1) or LOW (0V/0). Used to read buttons, switch states, or digital sensor outputs (like MQ-2 DO pin).

⬡ Arduino
digitalRead
int buttonState = digitalRead(2); // read pin 2 → HIGH or LOW if (buttonState == HIGH) { digitalWrite(13, HIGH); // LED ON if button HIGH } else { digitalWrite(13, LOW); // LED OFF otherwise } // Also works for MQ-2 DO pin: int gasAlert = digitalRead(2); // LOW = gas detected (above threshold)

3. analogRead(pin) — Read an analog sensor value

📊
What it does

Reads the analog voltage on a pin (A0–A5) and converts it to a number between 0 and 1023 (10-bit ADC). 0 = 0V, 1023 = 5V. Used to read the MQ-2 AO pin for gas concentration.

⬡ Arduino
analogRead
int sensorValue = analogRead(A0); // reads A0: 0 (0V) to 1023 (5V) // Convert raw value to voltage: float voltage = sensorValue * (5.0 / 1023.0); Serial.print("Raw: "); Serial.println(sensorValue); Serial.print("Volts: "); Serial.println(voltage);

4. pinMode(pin, mode) — Configure a pin

🔧
What it does

Sets a pin as either INPUT (to read a signal) or OUTPUT (to write/drive a signal). Must be called in setup() before using digitalWrite() or digitalRead().

⬡ Arduino
pinMode modes
pinMode(13, OUTPUT); // pin 13 will SEND voltage (LED) pinMode(2, INPUT); // pin 2 will READ voltage (button/sensor) pinMode(2, INPUT_PULLUP); // pin 2 + internal pull-up resistor (default=HIGH)

5. delay(ms) — Pause the program

⬡ Arduino
delay examples
delay(1000); // pause 1000 ms = 1 second delay(500); // pause 500 ms = 0.5 seconds delay(2000); // pause 2000 ms = 2 seconds delay(20000); // pause 20 seconds (MQ-2 warmup)

INPUT_PULLUP — The Inverted Logic!

⚡
CRITICAL — Logic is INVERTED with INPUT_PULLUP

When you use INPUT_PULLUP, the internal 50kΩ resistor pulls the pin to 5V by default.
• Button NOT pressed → pin reads HIGH
• Button PRESSED → pin reads LOW (because it's connected to GND)
This is the OPPOSITE of what you might expect!

⬡ Arduino
INPUT_PULLUP example
pinMode(2, INPUT_PULLUP); // pin 2 with internal pull-up int val = digitalRead(2); if (val == LOW) { // button IS pressed! (LOW because pullup pulls to HIGH when not pressed) digitalWrite(13, HIGH); // turn LED ON } else { digitalWrite(13, LOW); // LED OFF }

📡 Serial Communication

Serial.begin(9600) — Initializes serial communication at 9600 baud (bits/second). Must be called in setup(). Opens communication channel between Arduino and the computer's Serial Monitor.

⬡ Arduino
Serial print example
void setup() { Serial.begin(9600); // MUST be in setup(), baud=9600 } void loop() { Serial.print("Value: "); // prints WITHOUT new line Serial.println(42); // prints number WITH new line Serial.println("Hello!"); // prints string WITH new line delay(1000); }
🔗
UART Cross-Connection Rule (exam answer)

When Arduino connects to another serial device:
Arduino TX (Pin 1) → Device's RX pin
Arduino RX (Pin 0) → Device's TX pin
In Proteus simulation: Connect Arduino RX to Virtual Terminal's TXD.

💨 MQ-2 Gas Sensor — Pin by Pin

The MQ-2 is a Metal Oxide Semiconductor (MOS) gas sensor. It can detect LPG, Methane, CO, Alcohol, Smoke, H2, Propane — but cannot identify which gas is present. Range: 200–10,000 ppm.

MQ-2 Sensor Pin Diagram

PinNameDescriptionConnect ToUse
VCCPower5V power supply (±0.1%)Arduino 5VPowers the sensor heater element
GNDGround0V referenceArduino GNDCompletes the circuit
AOAnalog OutputVoltage proportional to gas ppm (0V=clean, 5V=max gas)Arduino A0analogRead(A0) → 0–1023 value
DODigital OutputHIGH/LOW based on threshold (potentiometer-set)Arduino D2digitalRead(2) → HIGH or LOW

Two Modes of Reading MQ-2

📊 Analog Mode (AO pin)

Use analogRead(A0) → returns 0–1023.
0 = clean air (0V), 1023 = maximum gas concentration (5V).
Use for precise ppm calculations.

🔲 Digital Mode (DO pin)

Use digitalRead(2) → returns HIGH or LOW.
LOW = gas exceeds threshold (alert!)
HIGH = gas below threshold (safe).
Threshold set by blue potentiometer on sensor.

⏰
Pre-Heat Time — Very Important!

MQ-2 requires at least 30 minutes (ideally 1 hour) warm-up for accurate readings. In code: add delay(20000) in setup() as minimum. The sensor has a small heating coil that must reach operating temperature.

MQ-2 Full Code Example

⬡ Arduino C++
MQ-2 analog reading
const int sensorPin = A0; // MQ-2 AO pin → Arduino A0 const int alertPin = 2; // MQ-2 DO pin → Arduino D2 const int ledPin = 13; // LED on D13 void setup() { Serial.begin(9600); pinMode(sensorPin, INPUT); pinMode(alertPin, INPUT); pinMode(ledPin, OUTPUT); Serial.println("MQ-2 warming up..."); delay(20000); // 20 second minimum warmup } void loop() { int rawValue = analogRead(sensorPin); // 0-1023 (AO pin) int alert = digitalRead(alertPin); // HIGH/LOW (DO pin) Serial.print("Gas raw: "); Serial.println(rawValue); if (alert == LOW) { // LOW = threshold exceeded! digitalWrite(ledPin, HIGH); // alert LED ON Serial.println("GAS DETECTED!"); } else { digitalWrite(ledPin, LOW); // LED OFF, safe } delay(500); }

🖥️ OLED Display (SSD1306)

The SSD1306 128×64 OLED works via I2C protocol. I2C uses only 2 wires (SDA + SCL) to communicate, making it very efficient. The OLED address is usually 0x3C.

OLED Wiring to Arduino

OLED Pin → Arduino Pin Reason VCC → 5V Power supply GND → GND Ground reference SDA → A4 I2C DATA line SCL → A5 I2C CLOCK line

Required Libraries (install from Library Manager)

  • Adafruit_SSD1306.h — main OLED driver
  • Adafruit_GFX.h — graphics primitives
  • Wire.h — I2C communication (built-in)
⬡ Arduino C++
OLED + MQ-2 combined display
#include <Wire.h> #include <Adafruit_GFX.h> #include <Adafruit_SSD1306.h> #define SCREEN_ADDRESS 0x3C // find with i2c_scanner! Adafruit_SSD1306 display(128, 64, &Wire, -1); void setup() { Serial.begin(9600); if (!display.begin(SSD1306_SWITCHCAPVCC, SCREEN_ADDRESS)) { Serial.println("OLED not found!"); while(1); // halt forever if OLED fails } display.clearDisplay(); display.setTextColor(WHITE); display.setTextSize(1); display.display(); } void loop() { int gasVal = analogRead(A0); // read MQ-2 display.clearDisplay(); // clear screen first! display.setCursor(0, 0); // position: x=0, y=0 display.println("Gas Monitor"); display.setCursor(0, 16); // move down 16 pixels display.print("Value: "); display.println(gasVal); display.display(); // MUST call this to show! delay(500); }
🔍
Finding I2C Address

Go to: File → Examples → Wire → i2c_scanner. Upload and open Serial Monitor. The address (0x3C or 0x3D) will be printed. Use this address in your OLED code.

🔌 Full System Wiring

==== MQ-2 Gas Sensor → Arduino ==== MQ-2 VCC → Arduino 5V (5V required for heater) MQ-2 GND → Arduino GND MQ-2 AO → Arduino A0 (analog 0-1023) MQ-2 DO → Arduino D2 (digital alert) 10K Resistor → Between DO and VCC (pull-up) ==== OLED Display (I2C) → Arduino ==== OLED VCC → Arduino 5V OLED GND → Arduino GND OLED SDA → Arduino A4 (I2C Data) OLED SCL → Arduino A5 (I2C Clock) ==== LED → Arduino ==== LED Anode (+) → 220Ω → D13 LED Cathode (-) → GND

Button controlling LED (with INPUT_PULLUP)

⬡ Arduino C++
Button + LED

🧠 Quick Quiz

Q1. What does digitalWrite(13, HIGH) do on Arduino Uno?
digitalWrite(13, HIGH) → sets pin 13 to 5V. On Arduino Uno, HIGH = 5V. ✓
Q2. Which function is used to read the MQ-2's analog gas concentration from pin A0?
analogRead(A0) reads the analog voltage → returns 0–1023. Used for continuous gas concentration. ✓
Q3. Which Arduino pins are used for I2C communication with the OLED display?
A4 = SDA (I2C Data), A5 = SCL (I2C Clock). These are the hardware I2C pins on Arduino Uno. ✓
Q4. What happens to the setup() function after the Arduino is powered on?
setup() runs ONCE when powered on, then loop() repeats forever. This is the fundamental Arduino program flow. ✓
Q5. When using INPUT_PULLUP on a button pin, what does it return when the button is pressed?
INPUT_PULLUP inverts the logic! Button pressed = LOW (connected to GND). Not pressed = HIGH (pulled to 5V by internal resistor). ✓
