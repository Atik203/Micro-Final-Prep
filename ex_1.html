🤖 Arduino Introduction

Arduino is an open-source hardware and software platform for building digital devices. The Arduino Uno is the most popular board for mid-level projects — it's what we use in this course.

💡
Key Fact

Arduino Uno uses the ATmega328P microcontroller running at 16 MHz with 2 KB RAM and 32 KB flash memory.

Arduino Applications

  • Home automation & smart systems
  • Robotics & motor control
  • IoT & environmental monitoring
  • Security systems
  • Weather monitoring & smart agriculture
  • Healthcare monitoring

📌 Pin Overview

The Arduino Uno has two types of pins: Digital and Analog.

Pin TypePinsVoltage LevelsFunction
Digital D0 – D13 HIGH = 5V, LOW = 0V Digital input or output — reads/writes HIGH or LOW only
Analog Input A0 – A5 0V to 5V range 10-bit ADC resolution (values 0–1023). Reads analog sensors.
UART TX Pin 1 (TX) Digital Transmit data via serial. Connect to RX of another device.
UART RX Pin 0 (RX) Digital Receive data via serial. Connect to TX of another device.
PWM 3, 5, 6, 9, 10, 11 0–255 (analogWrite) Pulse Width Modulation — simulate analog output
I2C SDA A4 Digital I2C data line — used by OLED, sensors
I2C SCL A5 Digital I2C clock line — used by OLED, sensors
Power 5V, 3.3V, GND — Power supply pins for sensors and components

⚠️ Exam Alert: When using UART serial communication, the Arduino's TX pin connects to the RX pin of another device, and vice versa. (TX ↔ RX cross-connection)

⚙️ setup() and loop() Functions

Every Arduino program requires exactly two functions:

  • Called once when the Arduino powers on or is reset
  • Used to initialize variables, pin modes, and configurations
  • Use pinMode(), Serial.begin() here
  • Think of it as your "one-time setup" block
⬡ Arduino C++
setup()
void setup() { pinMode(13, OUTPUT); // configure pin 13 as output Serial.begin(9600); // start serial at 9600 baud // runs only once! }
  • Runs continuously after setup() completes
  • This is where your main logic lives
  • Repeats as long as the Arduino is powered on
  • Used to: read sensors, control outputs, implement functionality
⬡ Arduino C++
loop()
void loop() { digitalWrite(13, HIGH); // LED ON delay(1000); // wait 1 second digitalWrite(13, LOW); // LED OFF delay(1000); // wait 1 second // repeats forever! }

📝 Key Arduino Functions (Syntax)

FunctionSyntaxWhat it does
pinMode pinMode(pin, mode) Sets a pin as INPUT or OUTPUT. e.g. pinMode(13, OUTPUT)
digitalWrite digitalWrite(pin, value) Sets digital pin HIGH (5V) or LOW (0V). e.g. digitalWrite(1, HIGH)
digitalRead digitalRead(pin) Reads a digital pin — returns HIGH or LOW. e.g. digitalRead(2)
analogRead analogRead(pin) Reads analog voltage (0–1023, 10-bit). e.g. analogRead(A0)
delay delay(ms) Pauses execution for specified milliseconds. e.g. delay(1000) = 1 second
Serial.print Serial.print(data) Prints to serial monitor (same line)
Serial.println Serial.println(data) Prints to serial monitor (new line)
INPUT_PULLUP pinMode(2, INPUT_PULLUP) Uses internal 50kΩ pull-up resistor. Default=HIGH, LOW when button pressed.

LED Blink Example — The Classic!

⬡ Arduino C++
LED Blink (Pin 1)
void setup() { pinMode(1, OUTPUT); // set pin 1 as output } void loop() { digitalWrite(1, HIGH); // turn LED ON (5V) delay(1000); // wait 1 second digitalWrite(1, LOW); // turn LED OFF (0V) delay(1000); // wait 1 second }

UART Cross-Connection Rule

Arduino TX (Pin 1) → Other device's RX pin
Arduino RX (Pin 0) → Other device's TX pin

💨 MQ-2 Gas Sensor

The MQ-2 is a Metal Oxide Semiconductor (MOS) type sensor. It detects multiple gases (LPG, Methane, CO, Alcohol, Smoke, H2, Propane) but cannot identify which gas is present. Range: 200–10,000 ppm.

MQ-2 Pin Diagram

PinNameDescriptionConnect To
VCCPower5V supply (±0.1%)Arduino 5V
GNDGroundGround reference (0V)Arduino GND
AOAnalog OutputAnalog voltage proportional to gas concentrationArduino A0 (analog pin)
DODigital OutputHIGH/LOW based on threshold set by potentiometerArduino Digital Pin 2

Operating Modes

📊 Analog Mode (AO pin)

Output voltage is proportional to gas concentration. Clean air → 0V. Max concentration (≥10,000 ppm) → 5V. Connect to Arduino A0 and use analogRead().

🔲 Digital Mode (DO pin)

Output is HIGH or LOW based on a threshold set by onboard potentiometer. Gas exceeds threshold → LOW output; below threshold → HIGH. Connect to digital pin.

⏰
Pre-Heat Time

MQ-2 requires at least 30 minutes (ideally 1 hour) to warm up for accurate readings. Add delay(20000) in setup() for code-level warmup.

PPM (Parts Per Million)

PPM measures gas concentration. 2 ppm methane means 2 out of every 1 million air molecules is methane. The MQ-2 sensor range is 200–10,000 ppm.

MQ-2 Arduino Code (Reading PPM)

⬡ Arduino C++
MQ-2 Gas Sensor Reading
const int sensorPin = A0; // analog pin #define RL 10 // load resistance (10 ohm) #define Ro 20 // resistance in fresh air #define m -0.263 // slope #define b 0.42 // intercept void setup() { Serial.begin(9600); Serial.println("MQ2 warming up!"); delay(20000); // allow MQ2 to warm up pinMode(sensorPin, INPUT); } void loop() { float sensorValue = analogRead(sensorPin); // 0-1023 float VRL = sensorValue * (5.0/1023.0); // convert to 0-5V float Rs = ((5.0*RL)/VRL) - RL; // sensor resistance float ratio = Rs/Ro; float ppm = pow(10, ((log10(ratio)-b)/m)); // calculate ppm Serial.print("PPM: "); Serial.println(ppm); }

🖥️ OLED Display (SSD1306)

The SSD1306 128×64 OLED works as an I2C device. It has a screen address (commonly 0x3C) that must be configured.

OLED Wiring to Arduino

OLED Pin → Arduino Pin VCC → 5V GND → GND SDA → A4 (I2C Data) SCL → A5 (I2C Clock)

Required Libraries

  • Adafruit_SSD1306.h — main OLED driver
  • Adafruit_GFX.h — graphics library
  • Wire.h — I2C communication

OLED "Hello World" Code

⬡ Arduino C++
OLED Display
#include <Wire.h> #include <Adafruit_GFX.h> #include <Adafruit_SSD1306.h> #define OLED_RESET -1 #define SCREEN_ADDRESS 0x3C // change to your I2C address! Adafruit_SSD1306 display(128, 64, &Wire, OLED_RESET); void setup() { if (!display.begin(SSD1306_SWITCHCAPVCC, SCREEN_ADDRESS)) { for (;;); // loop forever if OLED fails } display.clearDisplay(); display.setTextColor(WHITE); display.setTextSize(1); display.display(); } void loop() { display.clearDisplay(); display.setCursor(0, 12); display.println("Hello World!"); display.display(); delay(100); }
🔍
Finding I2C Address

Go to: Arduino IDE → File → Examples → Wire → i2c_scanner. Upload it and open the Serial Monitor to find your OLED's address (usually 0x3C or 0x3D).

🔌 Full System Wiring

Complete connection for MQ-2 Gas Sensor + OLED Display + LED with Arduino Uno:

==== MQ-2 Gas Sensor ==== MQ-2 VCC → Arduino 5V MQ-2 GND → Arduino GND MQ-2 AO → Arduino A0 (analog reading) MQ-2 DO → Arduino D2 (digital threshold) 10K Resistor between MQ-2 DO and VCC (pull-up) ==== OLED Display (I2C) ==== OLED VCC → Arduino 5V OLED GND → Arduino GND OLED SDA → Arduino A4 (I2C data) OLED SCL → Arduino A5 (I2C clock) ==== LED ==== LED (+) → Digital Pin 13 (through resistor) LED (-) → GND

💻 More Code Examples

Push Button Controlling LED

⬡ Arduino C++
Push Button + LED
void setup() { Serial.begin(9600); pinMode(2, INPUT_PULLUP); // pin 2 = button (internal pull-up) pinMode(13, OUTPUT); // pin 13 = LED } void loop() { int sensorVal = digitalRead(2); // HIGH when not pressed Serial.println(sensorVal); // With INPUT_PULLUP: logic is INVERTED // Button pressed = LOW, not pressed = HIGH if (sensorVal == LOW) { digitalWrite(13, HIGH); // LED ON when button pressed } else { digitalWrite(13, LOW); // LED OFF when released } }
⚡
INPUT_PULLUP — Logic is INVERTED!

When using INPUT_PULLUP, the default state is HIGH. When the button is pressed, it reads LOW. This is the opposite of what you might expect!

🧠 Quick Quiz — Test Yourself

Q1. What is the voltage level for a HIGH state signal in digital pins on the Arduino Uno board?
HIGH state = 5V on Arduino Uno ✓
Q2. When using UART on Arduino Uno, which pin should connect to the TX of another device?
RX of Arduino connects to TX of the other device (cross-connection) ✓
Q3. What does delay(1000) do in Arduino code?
delay() takes milliseconds — delay(1000) = 1 second ✓
Q4. Which OLED pins connect to Arduino A4 and A5?
A4 = SDA (I2C data), A5 = SCL (I2C clock) ✓
