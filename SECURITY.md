# Security Policy

We take security and academic integrity seriously. Please review this policy to report any security vulnerabilities or privacy concerns.

---

## Supported Versions

Only the latest version of the study guide is supported. Security fixes will be released on the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

---

## Reporting a Vulnerability

If you discover a security vulnerability (such as a database credential exposure in sample codes or cross-site scripting flaws), **please do not open a public issue.**

Instead, report it directly to the repository maintainers:
1. Email your report to the maintainer's contact details listed on their GitHub profile.
2. Include a description of the vulnerability, steps to reproduce, and any suggested patch.
3. Allow up to 48 hours for a response before follow-up.
