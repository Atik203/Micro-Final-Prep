Experiment 1: Arduino + MQ-2 Gas Sensor + OLED

Arduino Uno interfacing with MQ-2 gas sensor for gas detection and SSD1306 OLED display for data visualization.

📦 Component Checklist (click to check off)

Arduino Uno
MQ-2 Gas Sensor
OLED Display (SSD1306)
LED
10K Ohm Resistor
Breadboard
Jumper Wires
USB Cable (Arduino)
Laptop + Arduino IDE

🔌 Wiring Diagram

MQ-2 Gas Sensor → Arduino
MQ-2 VCC → Arduino 5V (operating voltage: 5V)
MQ-2 GND → Arduino GND
MQ-2 AO → Arduino A0 (analog reading, 0–1023)
MQ-2 DO → Arduino D2 (digital threshold output)
10K Resistor → Between MQ-2 DO and VCC (pull-up)
OLED Display (I2C) → Arduino
OLED VCC → Arduino 5V
OLED GND → Arduino GND
OLED SDA → Arduino A4 (I2C Data)
OLED SCL → Arduino A5 (I2C Clock)
LED → Arduino
LED (+) Anode → Arduino D13 (through resistor)
LED (-) Cathode → Arduino GND

📋 Step-by-Step Connection Guide

1
Preheat the MQ-2 Sensor (30+ minutes)
Connect MQ-2 VCC and GND to Arduino's 5V and GND. Let it warm up for at least 30 minutes for accurate readings. The sensor has a small heating element inside. In code, add delay(20000) in setup() as a minimum warmup.
2
Connect MQ-2 Gas Sensor
VCC → Arduino 5V (operating voltage: 5V ±0.1%)
GND → Arduino GND
AO → Arduino A0 (for analog readings — ppm calculation)
DO → Arduino D2 (for digital threshold detection)
10K Resistor between DO pin and VCC (pull-up resistor)
3
Connect OLED Display (I2C)
VCC → Arduino 5V
GND → Arduino GND
SDA → Arduino A4 (I2C Data line)
SCL → Arduino A5 (I2C Clock line)
Note: The OLED uses I2C protocol — two wires carry all data. Find I2C address using i2c_scanner example (usually 0x3C).
4
Connect LED
LED anode (+) → 220Ω resistor → Arduino D13
LED cathode (-) → Arduino GND
The resistor limits current to prevent burning the LED.
5
Install Libraries in Arduino IDE
Go to: Sketch → Include Library → Manage Libraries
Search and install: Adafruit SSD1306, Adafruit GFX, Adafruit BusIO
6
Upload Code and Test
Select the correct board (Arduino Uno) and COM port in Tools menu. Upload the code. Open Serial Monitor to see sensor readings. Check OLED display for output.
⚠️

Common Mistakes — Experiment 1

• Connecting OLED SDA to digital D4 instead of A4 — must use A4!
• Wrong I2C address in code (check with i2c_scanner first)
• Not adding MQ-2 warmup delay in setup()
• Forgetting the resistor between LED and Arduino pin

⚡ Quick Reference Code — Ex 1

⬡ Arduino C++
Gas Sensor + OLED Combined
#include <Wire.h> #include <Adafruit_GFX.h> #include <Adafruit_SSD1306.h> #define SCREEN_ADDRESS 0x3C // your OLED I2C address Adafruit_SSD1306 display(128, 64, &Wire, -1); const int sensorPin = A0; void setup() { Serial.begin(9600); Serial.println("MQ2 warming up!"); delay(20000); // warm up MQ2 if (!display.begin(SSD1306_SWITCHCAPVCC, SCREEN_ADDRESS)) while(1); // halt if OLED fails display.clearDisplay(); display.setTextColor(WHITE); display.setTextSize(1); display.display(); } void loop() { int sensorValue = analogRead(sensorPin); // 0-1023 display.clearDisplay(); display.setCursor(0, 0); display.print("Gas sensor: "); display.println(sensorValue); display.display(); Serial.print("Gas sensor: "); Serial.println(sensorValue); delay(500); }

Experiment 2: ESP32 + DHT11 + XAMPP

ESP32 connects to WiFi, reads DHT11 temperature/humidity data, and sends it to a local XAMPP server via HTTP POST.

📦 Component Checklist (click to check off)

ESP32 Dev Board
DHT11 Sensor
MQ-2 Gas Sensor
LED
10K Ohm Resistor
Breadboard
Jumper Wires
Micro-USB Cable
Laptop + XAMPP running
WiFi/Mobile Hotspot

🔌 Wiring Diagram

DHT11 Sensor → ESP32
DHT11 VCC → ESP32 3.3V (or 5V via VIN)
DHT11 GND → ESP32 GND
DHT11 DATA → ESP32 GPIO19 (D19) (#define DHTPIN 19)
10K Resistor → Between DATA and VCC (pull-up)
MQ-2 Gas Sensor → ESP32 (Optional for lab)
MQ-2 VCC → ESP32 VIN (5V)
MQ-2 GND → ESP32 GND
MQ-2 AO → ESP32 GPIO34 (input-only ADC pin)
LED → ESP32
LED (+) → ESP32 GPIO (via 220Ω resistor)
LED (-) → ESP32 GND

📋 Step-by-Step Connection Guide

1
Connect DHT11 to ESP32
VCC → ESP32 3.3V (can use 5V via VIN pin)
GND → ESP32 GND
DATA → ESP32 GPIO19
Add 10K pull-up resistor between DATA and VCC.
2
Set Up XAMPP on Laptop
1. Open C:\xampp\xampp-control.exe
2. Start Apache and MySQL
3. Verify: open browser → type localhost → XAMPP page should appear
4. Go to localhost/phpmyadmin → Create database + table
3
Create PHP File in htdocs
Create folder: C:\xampp\htdocs\dht11_project\
Create file: testdata.php
Write PHP code to connect to DB and INSERT received data.
4
Find Your Computer's IP Address
Open Windows Command Prompt (search "cmd")
Type: ipconfig
Note the IPv4 Address (e.g., 192.168.1.105)
The URL in ESP32 code will be: http://192.168.1.105/dht11_project/testdata.php
5
Upload ESP32 Code
Set your WiFi SSID and password in the code.
Set the URL with your laptop's IP address.
Upload. If upload fails: hold BOOT button + press EN button, then try again.
Open Serial Monitor (115200 baud) — should show connection status and response code 200.
6
Verify Data in phpMyAdmin
Go to localhost/phpmyadmin → select your database → click the table.
You should see temperature and humidity values being inserted every 5 seconds!
⚠️

Common Mistakes — Experiment 2

• ESP32 and laptop must be on the SAME WiFi network (use mobile hotspot if no lab WiFi)
• Using wrong laptop IP address in the URL (re-check with ipconfig)
• Apache and MySQL not started in XAMPP control panel
• Wrong Serial baud rate — ESP32 uses 115200, not 9600
• Not holding BOOT button during upload when it fails

⚡ Quick Reference Code — Ex 2

⬡ Arduino C++
ESP32 → XAMPP (key parts)
#include <WiFi.h> #include <HTTPClient.h> #include <DHT.h> #define DHTPIN 19 #define DHTTYPE DHT11 DHT dht11(DHTPIN, DHTTYPE); // ⚠️ Change these 3 values: String URL = "http://YOUR_IP/dht11_project/testdata.php"; const char* ssid = "YOUR_WIFI"; const char* password = "YOUR_WIFI_PASS"; void setup() { Serial.begin(115200); // ESP32 uses 115200! dht11.begin(); connectWiFi(); } void loop() { int t = dht11.readTemperature(); int h = dht11.readHumidity(); String postData = "temperature=" + String(t) + "&humidity=" + String(h); HTTPClient http; http.begin(URL); http.addHeader("Content-Type", "application/x-www-form-urlencoded"); int code = http.POST(postData); // 200 = success Serial.println(code); delay(5000); } void connectWiFi() { WiFi.mode(WIFI_STA); WiFi.begin(ssid, password); while(WiFi.status() != WL_CONNECTED) { delay(500); Serial.print("."); } Serial.println(WiFi.localIP()); }

Final Hardware Tips

1. Know pin connections by heart — no need to memorize every pin, just the main ones (SDA/SCL for OLED, AO for MQ-2, D19 for DHT11).
2. Verify circuit before powering — double-check VCC/GND to avoid damage.
3. Baud rate matters — Arduino=9600, ESP32=115200 for Serial Monitor.
4. BOOT button for ESP32 upload — remember to hold it if upload fails.
5. I2C address scan first for OLED — use i2c_scanner example before running OLED code.
