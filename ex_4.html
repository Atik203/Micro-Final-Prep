📷 Raspberry Pi Camera Modules

The Raspberry Pi Foundation offers several camera modules. They all connect via the CSI (Camera Serial Interface) port using a ribbon cable.

Original 5MP (2013)

OmniVision OV5647 sensor. Still photos up to 2592×1944. Video: 1080p30. Fixed-focus lens. Compatible with all Pi models.

Camera Module 2 — 8MP (2016)

Sony IMX219 sensor. Photos: 3280×2464. Video: 1080p30, 720p60, VGA90. Still the most common module.

Camera Module 3 — 12MP (2023)

Sony IMX477 sensor. Photos: 4056×3040. Video: 4K30, 1080p60. Autofocus lens — newest addition.

NoIR Version

Same specs as regular module but no infrared filter. Used with IR light source for night photography.

Global Shutter Camera

Captures all pixels simultaneously. Minimizes distortion for fast-moving objects. Uses smaller CSI connector.

High Quality Camera

12.3 MP Sony IMX477. Supports C and CS mount lenses for professional photography applications.

🔌 CSI Port & Camera Connection

The CSI port is located between the audio port and HDMI port on most Raspberry Pi models. All official camera modules use a ribbon cable to connect.

📸
Connection Steps

1. Power OFF the Raspberry Pi first.
2. Locate the Camera Module port (flat connector near HDMI).
3. Gently pull up on the plastic clip of the port.
4. Insert the ribbon cable — shiny contacts facing away from USB ports.
5. Push the plastic clip back down to secure the cable.

⚙️ Enable Camera Interface

After connecting the camera, you must enable it in Raspberry Pi settings:

Method 1: GUI

Go to Main Menu → Raspberry Pi Configuration → Interfaces tab → Camera: Enable

Method 2: Terminal (raspi-config)

pi@raspberrypi:~$ sudo raspi-config
# Then: Interface Options → Enable Camera → Yes → Finish → Reboot

💻 Command Line Tools

Two main command-line tools for camera control:

raspistill — Capture Images

$ raspistill -o Desktop/image.jpg
# Takes a photo (5 second preview then capture)
$ raspistill -o Desktop/image-small.jpg -w 640 -h 480
# Custom resolution: width=640, height=480

raspivid — Record Video

$ raspivid -t 10000 -o Desktop/video.h264
# Records 10 seconds (10,000 ms) at 1920×1080
▶️
Playing the Video

Double-click the video.h264 file on the Desktop to open it in VLC Media Player.

🐍 Python picamera Library

The picamera library allows full Python control of the Camera Module. File should be saved as camera.py (NOT picamera.py!).

Basic: Capture an Image

Python 3
camera.py — Capture photo
from picamera import PiCamera from time import sleep camera = PiCamera() camera.start_preview() # show preview (5 sec) sleep(5) # wait ≥2 sec before capture! camera.capture('/home/pi/Desktop/image.jpg') camera.stop_preview()
⚠️
Important Note

Always sleep for at least 2 seconds before capturing — this gives the camera sensor time to adjust to light levels. Camera preview only works when a monitor is connected (not over SSH/VNC).

Capture Multiple Photos (Loop)

Python 3
Take 5 photos in a loop
camera.start_preview() for i in range(5): # i = 0, 1, 2, 3, 4 sleep(5) camera.capture('/home/pi/Desktop/image%s.jpg' % i) # saves: image0.jpg, image1.jpg, ..., image4.jpg camera.stop_preview()

Record Video

Python 3
Record 5 second video
camera.start_preview() camera.start_recording('/home/pi/Desktop/video.h264') sleep(5) # record for 5 seconds camera.stop_recording() camera.stop_preview()

🎨 Camera Settings & Effects

Resolution & Framerate

Python 3
Max resolution photo
camera.resolution = (2592, 1944) # max still: 2592×1944 camera.framerate = 15 # required for max resolution # Max video: 1920×1080 | Min: 64×64

Rotate Image

Python 3
camera.rotation = 180 # rotate 180° (useful if preview is upside-down) # Can also use: 90, 270, or 0 (to reset)

Transparent Preview

Python 3
camera.start_preview(alpha=200) # alpha: 0 (invisible) to 255 (opaque)

Add Text Annotation

Python 3
camera.annotate_text = "Hello world!" camera.annotate_text_size = 50 # size: 6 to 160, default 32

Brightness & Contrast

Python 3
camera.brightness = 70 # 0-100, default 50 camera.contrast = 50 # 0-100, default 0

Image Effects (camera.image_effect)

none oilpaint blur negative hatch cartoon solarize sketch colorswap watercolor pastel emboss film posterise washedout denoise
Python 3
Apply effect
camera.image_effect = 'colorswap' # apply colorswap effect # Loop through ALL effects: for effect in camera.image_effects: camera.image_effect = effect camera.annotate_text = "Effect: %s" % effect sleep(5)

Exposure Modes

Use camera.exposure_mode. Options: off, auto, night, nightpreview, backlight, spotlight, sports, snow, beach, verylong, fixedfps, antishake, fireworks. Default: auto.

White Balance Modes

Use camera.awb_mode. Options: off, auto, sunlight, cloudy, shade, tungsten, fluorescent, incandescent, flash, horizon. Default: auto.

🤖 Object Detection with TensorFlow Lite

The Raspberry Pi's processing power can run ML algorithms! We use a pre-trained SSDLite-MobileNet-v2 model (from Google) to detect 80 common objects in real-time.

Tools Required

TensorFlow Lite

Lightweight version of TensorFlow optimized for mobile and embedded devices. Designed for resource-constrained platforms like Raspberry Pi.

OpenCV (cv2)

Open Source Computer Vision Library. Used to grab images from the Pi Camera video feed and draw detection results (bounding boxes, labels) on them.

Setup Steps

1

Update Raspberry Pi

Keep software current for compatibility.

sudo apt-get update && sudo apt-get dist-upgrade
2

Clone the Repository

Download scripts from EdjeElectronics GitHub repo.

git clone https://github.com/EdjeElectronics/TensorFlow-Lite-Object-Detection-on-Android-and-Raspberry-Pi.git
mv TensorFlow-Lite-Object-Detection... objDetection
cd objDetection
3

Create Virtual Environment

Isolated Python environment for dependencies (like a separate "room" for this project's software).

sudo pip3 install virtualenv
python3 -m venv objDetection-env
source objDetection-env/bin/activate
4

Install TensorFlow Lite + OpenCV

Run the shell script to install all dependencies (~400MB download).

bash get_pi_requirements.sh
sudo apt-get install libopenblas-dev
pip install numpy==1.26.4
5

Download & Deploy the Model

Download Google's SSDLite-MobileNet-v2 model trained on MSCOCO dataset (detects 80 objects).

wget https://storage.googleapis.com/.../coco_ssd_mobilenet_v1_1.0_quant.zip
unzip *.zip -d Sample_TFLite_model
6

Run Object Detection!

After a few moments of initialization, a window shows the camera feed with bounding boxes and labels on detected objects.

python3 TFLite_detection_webcam.py --modeldir=Sample_TFLite_model
💡
Virtual Environment Explained

A virtual environment is an isolated Python environment — like separate rooms in a house where each room has different furniture (libraries). Every time you open a new terminal, reactivate it: source objDetection-env/bin/activate

Common Errors & Fixes
